# Модернизация раздела «Сообщения» (Воронка продаж)

Файл: `client/src/pages/sales/InstagramMessagesPage.tsx` (2276 строк) + бэкенд-поддержка. Это inbox Instagram Direct с 3 колонками: список диалогов · тред · карточка лида.

## Что я выяснил (важно для плана)
- Вебхук **уже подписан** на `messaging_seen` (`server/services/instagram.ts:107`), но `processMessagingEvent` делает ранний `return` на `:1111` для всего, где нет `event.message` → события «прочитано/доставлено» приходят, но **тихо отбрасываются**. Это объясняет баг «всегда две галочки».
- В таблице `instagram_messages` есть колонка `status` (`sent`/`received`), но **нет** временных меток прочтения/доставки.
- Реальные typing-индикаторы от собеседника Instagram менеджеру **не приходят** (ограничение Graph API). Поэтому индикатор «печатает…» будет локальным (ваш собственный) + лёгкая эвристика по свежему входящему. Врать про чужой typing не буду.

## Часть A — Фронтенд: визуал + UX (вся колонка треда и список)

**1. Современные бабблы**
- Мягкие тени, аккуратные «хвостики», градиент для исходящих (`from-primary to-primary/90`), чистая карточка для входящих.
- Убрать устаревший точечный фон (`bg-[radial-gradient...]`) → спокойный `bg-muted/30`.

**2. Hover-панель действий сообщения** (замена кривому copy-значку)
- Десктоп: при наведении на баббл — компактная плавающая панель: **Ответить, Копировать** (для своих неудавшихся — **Повторить**).
- Мобил: панель появляется по тапу (решает баг «copy недоступен на тача»).
- «Ответить» вставляет цитату в composer (`> …`), фокусирует textarea.

**3. Правильные статусы прочтения** (синхрон с Частью B)
- В зависимости от `message.status`/`readAt`: ⏳ отправляется → ✓ отправлено → ✓✓ доставлено → ✓✓ (синие) прочитано.
- Убираю ложные «прочитано».

**4. Unread-разделитель в треде**
- При входе в диалог с непрочитанными — линия «Непрочитанные сообщения» (`t('unreadMessages')`) над первым непрочитанным сообщением. Требует от бэкенда «водяной знак» последнего прочитанного (см. Часть B — колонка `last_read_message_at`).

**5. Сортировка «непрочитанные сверху»**
- Тумблер в шапке списка диалогов (сохраняется в localStorage). При вкл: сначала непрочитанные по `lastInboundAt`, потом остальные по `lastMessageAt`.

**6. Мелкие UX-фиксы**
- Кнопка «Новые сообщения» — текст зависит от контекста (есть реально новые → «Новые сообщения», иначе → стрелка-вниз без текста/«К последнему»).
- Счётчик символов показывается только при `draft.length > 0`, мягкая подсветка у лимита.
- Кнопка «Повторить» при ошибке — полноценная кнопка под бабблом, не микро-ссылка.

## Часть B — Бэкенд: реальный трекинг прочтения

**1. Миграция БД** `migrations/0032_add_instagram_message_read_tracking.sql`
- `instagram_messages`: добавить `read_at timestamptz`, `delivered_at timestamptz`, индекс на `(conversation_id, read_at)`.
- `instagram_conversations`: добавить `last_read_message_at timestamptz` (водяной знак, для разделителя и сортировки).
- Обновить `shared/schema.ts` (`instagramMessages`, `instagramConversations`) + типы.

**2. Обработка вебхука** (`server/services/instagram.ts`, `processMessagingEvent`)
- Снять ранний `return` для событий без `message`. Добавить ветки:
  - `event.read` (`messaging_seen`): помечить все исходящие сообщения в диалоге до `event.read.watermark` как `read_at`, обновить `last_read_message_at`.
  - `event.delivery` (подпишемся дополнительно): `delivered_at` до watermark.
- Бродкаст: переиспользовать `INSTAGRAM_CONVERSATION_UPDATED` (расширив `data` флагом статуса), клиент инвалидирует кэш сообщений → бабблы перекрашиваются.

**3. Подписка на поле доставки**: добавить `message_deliveries` в `INSTAGRAM_WEBHOOK_FIELDS` (`:105`).

**4. Маршрут статуса**: при `sendInstagramTextMessage` статус остаётся `sent` → UI покажет ✓✓ доставлено после webhook'а. Без блокирующих запросов.

## Часть C — Индикатор «печатает…» (локальный + эвристика)

- **Ваш собственный** (честный): при наборе в composer шлём `INSTAGRAM_TYPING` через WS → у других менеджеров, открытых на этом диалоге, показывается «Менеджер печатает…».
- **Эвристика собеседника**: если последнее входящее сообщение новее ~20 сек и диалог открыт — короткая анимация «печатает…» (3 точки). Это подсказка, а не гарантия; в UI не утверждаем факт.
- Локальная анимация бабла-заглушки без новой БД-колонки (нет реальных typing-событий от IG).

## Риски / что отмечу
- typing собеседника — эвристика, не факт (оговорено выше).
- Для уже существующих сообщений `read_at` будет null → покажутся как «доставлено» (корректнее, чем нынешнее ложное «прочитано»); для новых заработает полностью.
- Все изменения обратной совместимы; миграция аддитивная (новые nullable-колонки).
