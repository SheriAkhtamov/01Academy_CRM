ОБЩАЯ ЦЕЛЬ

Создать единую систему, которая объединяет:

1. Маркетинговую воронку (от первого касания до оплаты)

2. Операционные процессы школы (расписание, посещаемость, прогресс, обратная связь)

3. Аналитику и финансы (LTV, окупаемость каналов, unit economics)

Система должна дать ответ на три вопроса в любой момент:

— Откуда пришёл каждый ученик и сколько он стоил?

— Как ученик прогрессирует и доволен ли он?

— Сколько зарабатывает школа с каждого канала, курса и группы?

1.4 Тёплая база (лиды со статусом «Не сейчас»)

Лиды, которые не конвертировались, не удаляются. Они переходят в тёплую базу и получают ежемесячную рассылку:

Частота: 1 раз в неделю — «Результат недели» (фото/видео проекта ученика)

Частота: 1 раз в 2 недели — приглашение на очередной демо-урок

Частота: 1 раз в месяц — специальное предложение (скидка / новый курс)

Система должна автоматически возвращать лида в активную воронку, если он откликнулся на рассылку.

2.3 Посещаемость

После каждого занятия преподаватель отмечает посещаемость (чеклист: был/не был).

Система должна:

— Автоматически считать % посещаемости каждого ученика

— Подсвечивать учеников с посещаемостью <70% (красный флаг)

— При 3 пропусках подряд: автоуведомление менеджеру → звонок родителю

— При посещаемости <50% за месяц: флаг «риск оттока»

2.4 Оценка удовлетворённости уроком (NPS ученика)

После каждого занятия ученику (или родителю для 7–10 лет) приходит короткий опрос:

Вопрос 1: «Как прошёл сегодняшний урок?» — шкала 1–5 (звёзды или эмодзи)

Вопрос 2: «Что понравилось больше всего?» — свободный текст (необязательно)

Вопрос 3: «Что можно улучшить?» — свободный текст (необязательно)

Канал отправки: WhatsApp или Telegram (ссылка на мини-форму)

Время отправки: через 30 минут после окончания занятия

Данные агрегируются:

— Средняя оценка по группе за неделю/месяц

— Средняя оценка по преподавателю

— Средняя оценка по курсу

— Тренд: улучшается/ухудшается

Если оценка <3: автоуведомление руководству + задача: связаться с учеником/родителем.

2.5 Обратная связь от родителей

Отдельно от оценки урока — ежемесячный опрос для родителей:

Вопрос 1: «Вы видите прогресс у ребёнка?» — Да / Скорее да / Скорее нет / Нет

Вопрос 2: «Ребёнок ходит на занятия с удовольствием?» — Да / Нет / По-разному

Вопрос 3: «Планируете продолжать в следующем месяце?» — Да / Скорее да / Не уверен / Нет

Вопрос 4: «Порекомендовали бы 01 Academy друзьям?» — шкала 0–10 (NPS)

Вопрос 5: «Комментарий» — свободный текст

NPS считается по стандартной формуле: (% промоутеров − % критиков).

Целевой NPS: >50.

Если родитель ответил «Не уверен» или «Нет» на вопрос 3 → автозадача менеджеру: позвонить и узнать причину.

3.2 LTV (Lifetime Value) — пожизненная ценность ученика

LTV = сумма всех оплат одного ученика за всё время обучения.

Система должна рассчитывать автоматически:

— LTV каждого ученика

— Средний LTV по курсу (AI Kids / AI Creator / Vibe Coding)

— Средний LTV по источнику (instagram_ad / referral / school_visit и т.д.)

— Средний срок обучения (в месяцах) по курсу

Целевые показатели:

AI Kids: LTV 4 800 000–6 000 000 сум (4–5 мес × 1 200 000)

AI Creator: LTV 8 640 000–10 080 000 сум (6–7 мес × 1 440 000)

Vibe Coding: LTV 10 000 000 сум (5 мес × 2 000 000)

3.3 CAC (Customer Acquisition Cost) — стоимость привлечения

CAC = общие расходы на маркетинг за период / количество новых оплативших учеников за тот же период.

Система должна считать:

— Общий CAC за месяц/квартал

— CAC по каждому каналу (Instagram ads / Telegram / TikTok / рефералы / школы и т.д.)

— CAC по каждому курсу

Для расчёта CAC по каналу нужно: расход на канал за период / количество оплативших учеников, пришедших с этого канала (по тегу источника).

Целевые показатели:

CAC < 300 000 сум (≈$23)

LTV:CAC > 10:1

3.4 Когортный анализ (Cohort Analysis)

Группировка учеников по месяцу первой оплаты. Отслеживание:

— Сколько из когорты продолжили во 2-й месяц? 3-й? 4-й?

— Retention rate по месяцам (% оставшихся)

— Revenue retention (доход от когорты по месяцам)

Это показывает реальный retention и помогает прогнозировать доход.

3.5 Окупаемость рекламных каналов (ROAS)

ROAS = доход от учеников с канала / расход на канал.

Система должна уметь:

— Связать каждого ученика с рекламным каналом (через тег источника)

— Суммировать все оплаты учеников с этого канала

— Разделить на расход (вводится вручную: «расход на Instagram ads за май = $300»)

— Показать ROAS за период

Целевой ROAS: >5x (на каждый потраченный доллар — $5 дохода).

РАЗДЕЛ 5: РЕФЕРАЛЬНАЯ СИСТЕМА

5.1 Механика

Каждый ученик (или родитель) получает уникальный реферальный код.

Код может быть: текстовым (например, TIMUR2026) или QR-кодом.

Когда новый лид приходит по рефералу:

— В карточке лида ставится тег referral_[имя]

— После оплаты нового ученика → автоматически:

  — Рефереру начисляется скидка 15% на следующий месяц

  — Новый ученик получает скидку 15% на первый месяц

Система отслеживает:

— Сколько рефералов привёл каждый ученик

— Сколько из них оплатили

— Общая сумма «заработанных» скидок

Уровни:

1 реферал → скидка 15% на 1 месяц

3 реферала → бесплатный месяц

5+ рефералов → статус «AI Ambassador» + бесплатное обучение
