# CRM Telegram Bot — доска задач 01Academy

Бот микроменеджмента доски CRM: публикует сводки задач в общей группе сотрудников.

## Что делает
- **10:00** (Пн–Пт, Asia/Tashkent) — утренняя сводка
- **16:00** (Пн–Пт, Asia/Tashkent) — дневная сводка

В каждой сводке:
- 🔴 **Просроченные** задачи (с числом дней просрочки)
- ⏰ **Дедлайн сегодня** — задачи «В процессе», у которых срок истекает сегодня
- ✅ **Выполнены, ждут принятия** — задачи в «Done» (с указанием, кто должен принять — постановщик)
- 📋 **To-Do по сотрудникам** — кто что должен взять в работу

Данные читаются напрямую из БД CRM (таблицы `board_tasks`, `users`), только на чтение.

## Запуск (локально)
```bash
npm install
cp .env.example .env   # заполнить BOT_TOKEN и GROUP_CHAT_ID
npm run summary:dry    # проверить текст сводки без Telegram (читает из БД)
npm run start          # запустить бота (polling + расписание)
```

## Настройка Telegram
1. Создать бота у [@BotFather](https://t.me/BotFather), получить `BOT_TOKEN`.
2. Добавить бота в рабочую группу.
3. В группе отправить `/chatid` — бот ответит id чата → вписать в `GROUP_CHAT_ID` (у групп он отрицательный).
4. `npm run start`.

## Команды бота
- `/summary` — показать сводку прямо сейчас (только в настроенной рабочей группе)
- `/chatid` — узнать id чата; без `GROUP_CHAT_ID` доступна для первичной настройки
- `/start` — справка (только в настроенной рабочей группе)

После заполнения `GROUP_CHAT_ID` бот отклоняет команды вне рабочей группы и не
раскрывает данные доски в личных или посторонних чатах. Длинные сводки
автоматически разбиваются на несколько сообщений в пределах лимита Telegram.

## Конфигурация (.env)
| Переменная | Назначение |
|---|---|
| `BOT_TOKEN` | токен от BotFather |
| `GROUP_CHAT_ID` | id группы для сводок |
| `DATABASE_URL` | подключение к Postgres CRM (read-only достаточно) |
| `BOARD_URL` | ссылка на доску в сводке |
| `TZ` | таймзона расписания (по умолчанию Asia/Tashkent) |
